## What's Actually Different Here

Most antidetect browsers spoof fingerprints at the JavaScript layer. Which sounds fine until you realize that modern anti-fraud systems on Facebook, Google, TikTok, and Amazon are specifically looking for JavaScript-level spoofing — it leaves traces, and those traces get your accounts linked.

Octo Browser does it at the **Chromium kernel level**. That's a fundamentally different approach. Each Octo profile is an isolated browser environment with its own:

- **Digital fingerprint** — over 50 configurable parameters including user agent, WebGL, screen resolution, hardware specs, OS signature
- **Cookie storage** — completely separate, with a built-in Cookie Robot to collect cookies automatically from target sites
- **Proxy assignment** — supports HTTP, HTTPS, SOCKS5, and SSH proxies, with a built-in Proxy Manager and Proxy Shop for buying residential IPs directly in the browser (discounts up to 60% for volume)
- **Activity history and tags** — so managing hundreds of profiles doesn't become chaos

Profile creation takes one click with auto-settings, or you can go manual and configure all 50+ parameters yourself. Either way, the profile is transferable — you can export the full environment (fingerprint, cookies, settings) and hand it to a teammate, who imports it and gets an identical setup with no drift.

API access (available from the Base plan up) lets you automate profile creation, configuration, and deletion using Puppeteer, Selenium, or CDP — without needing to launch the client.

**Platform support**: Windows, macOS (including M1 chips), and Linux. The team tracks Chromium updates proactively and pushes fast kernel updates, which keeps detection risk low as platforms evolve their anti-fraud systems.
